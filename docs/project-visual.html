Flood Risk Tool V4 Cleanup Map

A visual snapshot of what changed: the app still has the same homeowner assessment flow, but the project is now organized around clearer frontend modules, shared backend utilities, and documented legacy routes.

Product Flow

Assessment Form

Visitor enters contact, property, condition, and insurance details.

Report API

/api/flood-risk-report generates a structured risk report.

Risk Results

Frontend shows score, breakdowns, financial impact, and recommendations.

Lead Upsert

/api/lead/upsert saves, classifies, emails, and can sync HubSpot.

Nurture

Queued leads are processed through protected email follow-up.

What Moved
  • Embedded component CSS moved to src/styles.
  • Pure scoring and routing helpers moved to src/lib.
  • Reusable UI pieces moved out of the main component.
  • Shared Supabase and Resend setup moved to api/_lib.
What Stayed Primary
  • FloodRiskToolV4.js remains the main app screen.
  • /api/flood-risk-report remains the report endpoint.
  • /api/lead/upsert remains the lead workflow endpoint.
  • /api/nurture/process remains the scheduled email processor.
Health Check
  • npm test -- --watchAll=false passes.
  • npm run build compiles successfully.
  • Mojibake encoding was repaired.
  • Backup exists outside the repo.
Documentation Added

The stock Create React App README was replaced with project-specific setup, API, deployment, and maintenance notes. A new .env.example documents the expected configuration without storing secrets.
